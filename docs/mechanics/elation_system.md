# 欢愉命途机制


### 8.1 欢愉伤害

欢愉命途（Path of Elation）角色的主要输出手段为**欢愉伤害**，其伤害公式与常规伤害不同：

```
欢愉伤害 = 基础乘区(abilityMulti) × 欢愉度(elationMulti) × 笑点(punchlineMulti)
           × 暴击(critMulti) × 防御(defMulti) × 抗性(resMulti)
           × 易伤(vulnMulti) × 真实伤害(trueDmgMulti) × 特殊乘区(specialMulti)
```

> 英文术语对照：欢愉度 = **Elation**，笑点 = **Punchline**，好活当赏 = **Certified Banger**，阿哈时刻 = **Aha Instant**，欢愉伤害 = **Elation DMG**。

#### 欢愉度乘区（Elation Multiplier）

```
elationMulti = 1 + 欢愉度(Elation)
```

- **欢愉度（Elation）**：欢愉命途角色的专属面板属性，可通过角色行迹、光锥、辅助效果、星魂等途径提升
- 不同角色提升欢愉度的方式不同：
  - **爻光**：通过速度转模欢愉度
  - **火花**：通过攻击力转模欢愉度

#### 笑点乘区（Punchline Multiplier）

```
punchlineMulti = 1 + 5 × 笑点(Punchline) / (笑点 + 240)
```

等价形式：`6 - 1200 / (笑点 + 240)`，收敛到 **6**（即 +500% 上限）。

- **笑点（Punchline）**：全队共享的累积值，由欢愉角色技能和部分敌人/战斗效果产出
- 笑点越多，单个笑点的边际收益越低（稀释效应）
- **前 106 个笑点**：每个使 `punchlineMulti` 绝对值增加 **>0.01**
- **前 33 个笑点**：每个使欢愉伤害**相对增幅超过 1%**（此时 `punchlineMulti` 基数低，同样的绝对增量对总伤害贡献更大）

> 欢愉伤害**不享受增伤乘区**（即不吃我方角色提供的增伤 buff），**只享受敌方易伤**。
>
> 当伤害由「好活当赏（Certified Banger）」状态产生时，公式中的笑点改用该状态记录的笑点值计算（多次好活当赏的笑点合并计算）。

### 8.2 欢愉编号

欢愉角色拥有特有的**欢愉编号**，标识在角色的欢愉技中：

- 编号决定欢愉技在「阿哈时刻」中的**施放先后顺序**
- **编号越小，施放越靠前**
- 通常辅助角色的编号较靠前，以便提前覆盖战斗增益
- 可理解为欢愉体系内特有的"速度"或排列优先级

### 8.3 阿哈时刻（Aha Instant）

当队伍中存在欢愉角色时，行动条会基于全体欢愉角色速度一定比例生成**阿哈时刻**特殊单位。

#### 阿哈速度

```
阿哈速度 = 80 + V1 × 0.2 + V2 × 0.1 + V3 × 0.05 + V4 × 0.02
```

| 符号 | 说明 |
|------|------|
| 80 | 固定基础速度 |
| V1 | 队伍中最快的欢愉角色速度 |
| V2 | 第二快的欢愉角色速度 |
| V3 | 第三快的欢愉角色速度 |
| V4 | 第四快的欢愉角色速度 |

**举例**：爻光 200 速、银狼 180 速、花火 140 速

```
阿哈速度 = 80 + 200 × 0.2 + 180 × 0.1 + 140 × 0.05 = 145
```

#### 阿哈时刻运行机制

1. **进战后触发**：进战后爻光等角色可通过技能产出初始【笑点】，阿哈发现乐子后出现在行动轴中正常跑条
2. **笑点累积**：角色进行普攻、战技、大招等行为时，战技点上方会累积一定**阿哈笑点值**（全队共享）
3. **阿哈行动**：当阿哈行动时，开启「阿哈时刻」：
   - 消耗当前全部阿哈笑点值
   - 按**欢愉编号从小到大**的顺序，触发全队欢愉技（每个欢愉角色各施放一次欢愉技）
   - 造成大量欢愉伤害
   - 阿哈会总结刚才的总伤害量
4. **好活当赏（Certified Banger）**：阿哈时刻结束后，参与的角色获得**【好活当赏】BUFF**（持续 2 回合，每次独立计时）
   - 该状态记录本次阿哈时刻消耗的笑点值，多次好活当赏的笑点合并计算
   - 角色头像左下方显示【好活】增益层数
   - 【好活】增益使角色可以继续造成欢愉伤害，笑点值越高伤害越高
   - 不同角色的【好活】效果不同：
     - **爻光**：使我方角色的攻击附带欢愉伤害
     - **火花**：使自身部分技能额外造成欢愉伤害
5. **多波次保留**：在部分多波次战斗环境中，阿哈时刻**不用重新跑条**

> 角色行动 → 叠加阿哈笑点 → 阿哈行动触发全队欢愉技 → 阿哈笑点转变为角色笑点。
> 无论是阿哈还是角色，笑点越高造成的欢愉伤害越高。

### 8.4 体系特点

- **多核出伤**：欢愉体系鼓励多欢愉角色组队，提高笑点叠加效率，增加每次阿哈时刻发动的欢愉技数量
- **角色优先级 > 光锥星魂**：在规划抽取时，角色本身的优先级高于光锥和星魂
- **可拆分搭配**：部分情况下可拆分欢愉角色，搭配老角色的充能、易伤、减抗、穿透、真伤、拉条、双暴等辅助机制，组成 2~3 个队伍应对环境
- **欢愉伤害数字呈现彩色**，与其他伤害数字类型区分

### 8.5 角色示例机制

| 角色 | 欢愉度来源 | 特殊机制 |
|------|-----------|---------|
| **爻光** | 速度转模欢愉度 | 大招可直接开启 1 次额外的「阿哈时刻」；使我方角色攻击附带欢愉伤害 |
| **火花** | 攻击力转模欢愉度 | 为队伍产出额外的【笑点】；使自身部分技能额外造成欢愉伤害 |
