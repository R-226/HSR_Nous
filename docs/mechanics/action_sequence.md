# 行动序


### 3.1 行动值 (Action Value)

- 总路程：10000
- 初始行动值 = 10000 / 速度
- 所有单位（我方和敌方）同时消耗行动值，谁先归零谁先行动
- 行动值归零的角色获得行动机会
- 行动后立刻回到起点，获得新的初始行动值继续参与排序
- 若多个单位行动值相同，按队伍编排顺序排序

#### 轮次机制

| 玩法 | 0 轮行动值 | 后续轮次 | 说明 |
|------|-----------|---------|------|
| 忘却之庭 / 虚构叙事 | 150 | 100 | 标准模式 |
| 异相仲裁 | 300 | 100 | 长首轮模式 |

轮次特性：
- 轮次**不能被**推拉条、加速减速影响
- 轮次显示数字 = 剩余轮次数 - 1
- 轮次归 0 时战斗直接结束

角色每轮行动次数 = 轮次行动值 / 角色行动值

常见速度阈值：
- 首轮两动：10000 × 2 / 150 ≈ **133.3**
- 首轮三动：10000 × 3 / 150 = **200**

#### 战斗触发与行动推迟

| 触发方式 | 效果 |
|---------|------|
| 弱点进攻 | 造成弱点属性伤害和削韧，获取先手优势 |
| 正常进攻 | 无特殊效果 |
| 遇袭 | 我方角色行动推迟 **20** 点行动值 |
| 直接进入战斗（剧情/副本）| 无特殊效果 |

### 3.2 拉条与推条

#### 拉条（行动提前）

**立即行动**：直接将被拉条角色拉到行动条最顶端，当前行动值变为 **0**，无视当前行动值。

**行动提前 100%**：在当前行动值基础上减去按当前速度计算的 100% 提前量，即 `当前行动值 - 10000 ÷ 当前速度`。若此前角色被推条（行动值增加），提前后可能仍需跑剩余行动值。

**行动提前 N%（0 < N < 100）**：在当前行动值基础上减去按当前速度计算的提前量：

```
行动提前后行动值 = 当前行动值 - 10000 ÷ 当前速度 × 行动提前比例
```

- 结果小于 0 时取 0

#### 推条（行动延后）

```
行动延后后行动值 = 当前行动值 + 10000 ÷ 当前速度 × 行动延后比例
```

- 结果大于 999 时显示为 999

#### 后拉先动原则

当多个行动提前效果几乎同时生效，且导致多个目标行动值都为 0 时，**后变成 0 的单位先行动**。

例：B 被立即行动拉到最顶端，随后 C 被立即行动拉到最顶端，随后 E 被立即行动拉到最顶端，则行动顺序为 E → C → B。

### 3.3 速度变化

```
变化后速度 = 变化前速度 + 角色基础速度 × 加速百分比 + 固定值加速数值

变化后行动值 = 当前行动值 × 变化前速度 / 变化后速度
```

- 若当前行动值为 0，速度变化从**下一次**行动值计算开始生效
- 多回合持续的加速/减速：获得 buff 瞬间行动值按上述公式变化，后续均按变化后的速度计算初始行动值

### 3.4 额外回合与插入行动

以下行动为**插入式行动**，不消耗当前回合，不改变回合行动状态：

| 行动类型 | 说明 | 优先级 |
|---------|------|--------|
| 追加攻击 | 满足条件时自动触发 | 高 |
| 终结技 | 能量满时立即插队释放 | 中 |
| 命途回响 | 模拟宇宙等特殊机制 | 中 |
| 额外回合 | 特定角色机制（如希儿再现）| 中 |
| 战技触发 | 如青雀战技、刃战技 | 低 |

> 插入式行动**不消耗 buff 回合数**（因为不计入回合数）。
> 优先级为观察总结，实际以游戏程序逻辑为准。

### 3.5 冻结与强烈震荡

#### 冻结

被冻结的角色，**下一次行动时无法行动**，直接结算回合。解冻后获得补偿：

```
解冻后行动值 = 初始行动值 × 50%
```

> 若未跳过行动就被解除冻结，则不会触发补偿。

#### 强烈震荡

与冻结类似，被强烈震荡的角色下一次行动时无法行动，直接结算回合。解除后获得补偿：

```
解除后行动值 = 初始行动值 × 70%
```

> 若未跳过行动就被解除，则不会触发补偿。
